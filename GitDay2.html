üíª Installing Git

‚ÑπÔ∏è Before You Start

Git is a command-line tool that needs to be installed on your system. Choose your operating system below and follow the installation steps.

Installing Git on Windows

1Download the Installer

Visit the official Git website:

https://git-scm.com/downloads/win

The download will start automatically for the latest version of Git for Windows. Run the .exe file after download.

2Select Editor & Adjust Path

  • Choose your preferred text editor (Vim, Notepad++, or VS Code)
  • Add Git to your system PATH for command-line access
  • Select OpenSSL for secure HTTPS communication
  • Choose line ending conversion (Windows-style recommended)

3Complete Installation & Verify

After installation, open Git Bash or Command Prompt and run:

git --version

If Git is installed correctly, you'll see the version number displayed.

Installing Git on Linux

1Update System & Install

For Debian/Ubuntu:

sudo apt update
sudo apt install git

For Fedora:

sudo dnf install git

For Arch Linux:

sudo pacman -S git

2Verify Installation

git --version

Installing Git on macOS

1Install Homebrew

If you don't have Homebrew installed:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

2Install Git via Homebrew

brew install git

3Verify Installation

git --version

‚öôÔ∏è Configuring Git

‚ÑπÔ∏è Why Configure Git?

Git needs to know who you are so it can attribute your commits correctly. This information appears in your commit history.

Essential Configuration

1Set Your Name

git config --global user.name "Your Name"

2Set Your Email

git config --global user.email "youremail@example.com"

‚ö†Ô∏è Important

Use the same email address you'll use for GitHub to ensure proper commit attribution.

3Set Default Branch Name

git config --global init.defaultBranch main

4Verify Your Configuration

git config --list
# Or check specific values:
git config user.name
git config user.email

Optional But Useful Settings

Command Purpose
git config --global core.editor "code --wait" Set VS Code as default editor
git config --global color.ui auto Enable colored output
git config --global merge.conflictstyle diff3 Better merge conflict display

üîê Setting Up SSH Keys

‚ÑπÔ∏è Why SSH Keys?

SSH keys provide a secure way to authenticate with GitHub without entering your password every time. Highly recommended for daily use!

1Generate SSH Key

ssh-keygen -t rsa -b 4096 -C "youremail@example.com"

Press Enter to accept the default file location, and optionally enter a passphrase for extra security.

2Start SSH Agent & Add Key

# Start the SSH agent
eval "$(ssh-agent -s)"

# Add your SSH key
ssh-add ~/.ssh/id_rsa

3Copy Your Public Key

For Linux/Mac:

cat ~/.ssh/id_rsa.pub

For Windows (Git Bash):

clip < ~/.ssh/id_rsa.pub

4Add SSH Key to GitHub

  • Go to GitHub Settings ‚Üí SSH and GPG keys
  • Click "New SSH key"
  • Give it a descriptive title (e.g., "My Laptop")
  • Paste your public key
  • Click "Add SSH key"

5Test Your SSH Connection

ssh -T git@github.com

‚úÖ Success Message

You should see: "Hi username! You've successfully authenticated, but GitHub does not provide shell access."
